# L Health

Personal health tracker — daily water, exercise, body stats, cheat logs, and
goals — with user accounts and cloud sync. Next.js (App Router) deployed to
Cloudflare Workers, with Cloudflare D1 as the source of truth.

## Stack

- **Next.js 16** (App Router, React 19, TypeScript)
- **Tailwind CSS v4** + **shadcn/ui** (Radix) + **Recharts**
- **Better Auth** — email + password; sessions stored in D1
- **Cloudflare D1** + **Drizzle ORM**
- Deployed to **Cloudflare Workers** via **@opennextjs/cloudflare** (OpenNext)

## Prerequisites

- Node 20+ (developed on Node 26)
- A Cloudflare account for deployment (`wrangler login`)

## Local development

```sh
npm install
npm run db:migrate:local   # apply D1 migrations to the local (Miniflare) DB
npm run dev                # Next dev server at http://localhost:3000
```

Local secrets live in `.dev.vars` (copy `.env.example` and fill in
`BETTER_AUTH_SECRET`). `.dev.vars` is gitignored.

## Testing & checks

```sh
npm test          # Vitest — domain logic + D1-row → state mapping
npx tsc --noEmit  # type-check
npm run preview   # build + run the real Worker locally (workerd); use before deploy
```

## Database (Drizzle + D1)

- Schema: `src/db/schema.ts` (domain tables) + `src/db/auth.schema.ts`
  (generated by the Better Auth CLI).
- `npm run db:generate` → SQL migrations under `drizzle/migrations/`.
- Apply: `npm run db:migrate:local` / `npm run db:migrate:remote`.

## Project structure

- `src/app/` — routes: `/`, `/sign-in`, `/sign-up`, `/forgot-password`, `/reset-password`, `/dashboard`, `/privacy`, `/api/auth/[...all]`, `/api/export`
- `src/lib/domain/` — pure tracker logic (ported, unit-tested) + summaries
- `src/lib/actions.ts` — server actions (zod-validated, user-scoped writes)
- `src/lib/tracker-data.ts` — D1 read path → domain summaries for the dashboard
- `src/lib/auth.ts`, `src/lib/dal.ts` — Better Auth factory + session DAL
- `src/components/dashboard/` — dashboard UI sections
- `src/db/` — Drizzle schema + request-scoped client

## Deploy (Cloudflare)

```sh
wrangler login
wrangler d1 create l-health           # paste the returned database_id into wrangler.jsonc
npm run db:migrate:remote
wrangler secret put BETTER_AUTH_SECRET # also set BETTER_AUTH_URL to the deployed origin
npm run deploy
```
